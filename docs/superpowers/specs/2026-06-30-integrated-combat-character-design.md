# 一体化战斗角色与伪 3D 表现设计

## 目标

- 角色、双手和当前枪械必须形成一个完整、可信的战斗剪影，不再出现枪械悬浮在人旁边的视觉断裂。
- 保持现有俯视生存玩法、输入、数值和 HUD 行为不变，只改善战场呈现。
- 参考目标图的斜俯视废土气质，加强脚底锚点、落地阴影、后坐、枪口闪光和人物遮挡关系。

## 方案选择

采用“武器与人物合成的完整角色贴图 + 现有 Canvas 运动变换”。每张战斗贴图在资产阶段就包含双手、枪械和上身瞄准姿态；运行时只选择与当前枪械对应的贴图，并施加镜像、轻微压缩、步态起伏和后坐，不再把枪械图标单独贴在人物旁边。

不采用完整 3D 模型。完整 3D 需要模型、绑定、动画、材质和渲染管线迁移，超出本轮目标，也不是获得目标图观感的必要条件。

## 视觉与动作规则

- 基准姿态为斜俯视三分之四视角，双脚分开、身体压低、双手稳定持枪。
- 枪械资产按 `pistol`、`shotgun`、`smg`、`magnum` 映射到人物战斗贴图；进化武器沿用其基础枪械造型。
- 水平朝向通过镜像完成；垂直瞄准只影响有限幅度的身体倾斜和枪口光，不把整个人旋转成横躺姿态。
- 射击时人物围绕脚底锚点产生短促后坐，枪口在人物前方出现暖色闪光和短距离光束。
- 装备图标继续留在 HUD 和商店中，不再作为人物身边的悬浮小图标。
- 人物与敌人继续按脚底深度排序，阴影固定在地面，人物本体起伏不会带动阴影漂浮。

## 资产要求

- 新战斗角色贴图使用与现有角色一致的红围巾、橄榄色外套、战术背心和废土写实插画风格。
- 输出为透明 PNG，人物完整、边缘干净、无文字、无独立展示底座、无额外枪械或残肢。
- 武器必须由两只手实际握持，枪托/握把与手掌接触，枪身方向与头肩朝向一致。

## 代码边界

- 新增一个小型纯函数模块，负责把当前武器和瞄准输入转换为角色贴图键、镜像、倾斜和枪口位置，便于单元测试。
- `src/game.ts` 只消费该姿态结果并绘制角色、阴影和枪口反馈。
- 删除上一版独立枪械与随身装备贴图绘制路径；不修改武器伤害、射速、生成、碰撞或商店逻辑。

## 验收标准

1. 默认手枪状态下，人物与枪械在任何水平朝向都保持连成一体。
2. 切换霰弹枪、冲锋枪或马格南后，人物贴图随当前主武器变化。
3. 移动、射击、受击闪烁、护盾和脚底阴影仍正常。
4. 1280×720 实机截图中，人物不再出现悬浮枪械或悬浮装备图标，斜俯视层次和枪口反馈清晰。
5. 单元测试、类型检查、完整测试、构建和浏览器控制台检查全部通过。

## 非目标

- 本轮不引入 Three.js、WebGL 3D 模型、骨骼动画或八方向逐帧动画。
- 本轮不重做敌人资产、HUD 布局、地图生成或玩法数值。
